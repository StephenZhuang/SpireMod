该模组采用 **Gradle** 作为构建与依赖管理工具，针对《杀戮尖塔》（Slay the Spire）及其模组加载器 **ModTheSpire** 的特殊环境进行了定制。

### 1. 依赖声明与来源
- **核心依赖**：项目不依赖传统的 Maven/Gradle 中央仓库中的第三方库，而是直接依赖本地安装的游戏主程序 JAR (`desktop-1.0.jar`) 和模组加载器 JAR (`ModTheSpire.jar`)。
- **作用域**：这些依赖被声明为 `compileOnly`，意味着它们仅用于编译期类型检查和方法引用，不会被打包进最终的模组 JAR 文件中。这是模组开发的标准做法，因为游戏运行时环境中已经存在这些类。
- **仓库配置**：`build.gradle` 中保留了 `mavenCentral()` 仓库，但目前并未声明任何需要从该仓库下载的传递性依赖。

### 2. 版本管理与兼容性
- **模组元数据**：`src/main/resources/ModTheSpire.json` 明确指定了兼容的游戏版本 (`sts_version`: "12-18-2022") 和模组加载器版本 (`mts_version`: "3.30.0")。这充当了运行时的依赖版本约束。
- **Java 版本**：通过 `java.toolchain` 锁定使用 **Java 8**，确保与游戏原生的 JVM 环境兼容。

### 3. 本地化路径解析
- **动态路径**：`build.gradle` 定义了默认的系统特定路径（针对 macOS 的 Steam 目录结构），并允许通过 Gradle 属性 (`stsJar`, `mtsJar`, `modsDir`) 进行覆盖。这种设计使得不同操作系统的开发者可以通过命令行参数轻松适配各自的本地环境，而无需修改构建脚本。
- **构建验证**：在生成 JAR 包之前，任务会检查指定的游戏和模组加载器 JAR 文件是否存在，若缺失则抛出异常，防止因依赖缺失导致的无效构建。

### 4. 开发者规范
- **不要打包游戏类**：严禁将 `compileOnly` 依赖中的类打包进最终产物，否则会导致类加载冲突或模组体积异常增大。
- **环境适配**：在非 macOS 系统上开发时，必须通过 `-PstsJar=...` 等参数传入正确的本地路径，或修改 `build.gradle` 中的默认路径逻辑。
- **版本同步**：当游戏或 ModTheSpire 更新时，需同步更新 `ModTheSpire.json` 中的版本声明以及本地引用的 JAR 文件，以确保 API 兼容性。
