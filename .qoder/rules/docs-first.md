# 文档先行规则

所有功能变更（新增、修改、删除）必须遵循**文档先行**原则：

1. **先更新文档**：在修改代码之前，必须先更新 `docs/superpowers/specs/` 下的相关文档（PRD 和设计文档），确保文档反映最新的功能需求。
2. **再改代码**：文档确认无误后，再进行代码实现。
3. **同步更新**：如果实现过程中发现设计需要调整，必须先回头更新文档，再继续编码。

## 文档位置

- PRD：`docs/superpowers/specs/2026-06-17-spiremod-prd.md`（当前有效的完整功能文档）
- 设计文档：`docs/superpowers/specs/` 下按日期命名的设计文档
