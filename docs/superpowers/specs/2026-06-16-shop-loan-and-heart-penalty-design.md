# SpireMod 商店贷款与心脏惩罚设计

**日期**：2026-06-16  
**状态**：已实现  
**范围**：在现有轻量级原版遗物 Mod 基础上，增加商店贷款/还款机制，并在心脏战对未还贷款施加惩罚。

## 目标

本次改动包含五部分：

- 开局遗物移除 `Courier`，替换为 `Omamori`
- 商店主界面增加 `贷款` 按钮
- 若存在未偿还贷款，商店主界面增加 `还款` 按钮
- 最终幕商店禁止贷款
- 进入心脏战时，若仍有未偿还贷款，则施加 `商人的愤怒` 惩罚

## 设计原则

- 保持当前项目的极简风格，继续使用 `SpirePatch`
- 优先复用原版系统与原版 Power，减少额外资源和注册负担
- 新增功能尽量局限在商店界面、战斗前逻辑和本局运行时状态中
- 不引入 BaseMod，不做无关 UI 重构

## 玩家体验

### 开局

玩家仍然会获得现有的开局金币和遗物组合，但 `Courier` 改为 `Omamori`。

### 商店

- 在非最终幕商店中，出现一个 `贷款` 按钮（位于商店左上角）
- 每点击一次 `贷款`，玩家立刻获得 `100` 金币
- 累计贷款上限为 `500`
- 若已到上限，则按钮不再生效

### 还款

- 只要当前未偿还贷款大于 `0`，商店中出现 `还款` 按钮
- 每点击一次 `还款`，若玩家金币足够，则支付 `100` 金币并减少 `100` 贷款
- 若玩家金币不足 `100`，则本次还款失败，不修改贷款值

### 最终幕限制

- 当玩家当前地牢为 `TheEnding` 时，商店中不显示可用贷款入口
- 若之前已有贷款，仍允许在最终幕商店进行还款

### 心脏战惩罚

进入 `CorruptHeart` 战斗前，若未偿还贷款大于 `0`，则给予玩家 `商人的愤怒`

惩罚效果：

- 力量 `-10`
- 敏捷 `-10`
- 每回合失去 `10` 生命

## 技术设计

## 1. 贷款状态

新增一个轻量运行时状态类，例如 `LoanState`，用于维护本局贷款信息：

- `currentDebt`：当前未偿还贷款
- `LOAN_STEP = 100`
- `MAX_DEBT = 500`

提供方法：

- `reset()`
- `canBorrow()`
- `hasDebt()`
- `borrow(AbstractPlayer player)`
- `canRepay(AbstractPlayer player)`
- `repay(AbstractPlayer player)`

状态只在本次运行中有效，不额外做存档持久化。当前 mod 的其他功能同样偏向“新开局即时生效”，这次先保持一致。

## 2. 开局重置时机

为避免上一局残留，需在新开局初始化时重置贷款状态。可复用当前已存在的玩家初始化 patch 所处时机，在角色初始化阶段调用 `LoanState.reset()`。

## 3. 遗物替换

更新现有 `RelicPatch`：

- 删除 `Courier.ID`
- 增加 `Omamori.ID`

其余遗物保持不变。

## 4. 商店按钮

直接 patch `ShopScreen` 的以下入口：

- `open()`：初始化本次商店 UI 状态
- `update()`：处理按钮悬停、点击和逻辑执行
- `render(SpriteBatch)`：绘制按钮和贷款提示文本

### 按钮表现

为避免引入完整自定义 UI 系统，按钮采用轻量绘制方式：

- 使用 `Hitbox` 处理点击区域
- 使用 `FontHelper` 与 `ImageMaster.WHITE_SQUARE_IMG` 或等价轻量资源绘制按钮底色与文字
- 按钮风格保持简洁，优先保证可点击和可辨识

### 按钮逻辑

贷款按钮显示条件：

- 当前不是 `TheEnding`
- `currentDebt < 500`

还款按钮显示条件：

- `currentDebt > 0`

点击行为：

- `贷款`：调用 `LoanState.borrow(player)`
- `还款`：调用 `LoanState.repay(player)`

交互反馈：

- 成功贷款/还款后刷新商店提示语
- 金币不足、达到上限或不可贷款时，播放失败音效并给出短提示

## 5. 最终幕商店判定

直接以当前地牢实例或地牢 ID 判断：

- `AbstractDungeon.id`
- 或 `AbstractDungeon.getCurrRoom()` 所在地牢是否属于 `TheEnding`

实现上优先使用最稳定、最易读的判定方式，例如：

- `TheEnding.ID.equals(AbstractDungeon.id)`

## 6. 心脏战惩罚实现

patch `CorruptHeart.usePreBattleAction()` 或等价的战斗前入口。

当 `LoanState.hasDebt()` 为真时，对玩家施加：

- `StrengthPower(player, -10)`
- `DexterityPower(player, -10)`

“每回合失去 10 生命”优先考虑两种实现：

### 推荐实现

新增一个极小自定义 Power：

- `MerchantWrathPower`
- 回合开始时对拥有者造成 `10` 点生命流失或伤害
- Power 名称显示为 `商人的愤怒`

推荐原因：

- 名称和效果与需求完全一致
- 不需要挪用原版语义不符的 Power
- 代码量可控

### 非推荐实现

组合多个原版 Power 模拟持续掉血。

不推荐原因：

- 语义不准确
- 难以保证是“每回合 -10 生命”
- 玩家看到的效果名称也不符合需求

因此本次设计采用：

- 原版 `StrengthPower(-10)`
- 原版 `DexterityPower(-10)`
- 自定义 `MerchantWrathPower(10)`

## 7. 文案

本次改动至少需要以下文案常量：

- `贷款`
- `还款`
- `债务：X / 500`
- `已贷款 100 金币`
- `已还款 100 金币`
- `贷款已达上限`
- `金币不足，无法还款`
- `最终幕无法贷款`
- `商人的愤怒`

当前项目尚未建立完整本地化结构，因此本次先以内嵌常量形式实现，保持范围收敛。

## 8. UI 布局说明

贷款/还款按钮位于商店界面**左上角**，避免遮挡商店删牌区域（位于右侧）：

- `BUTTON_X = 140`，按钮中心 X 坐标
- `BORROW_Y = 940`，贷款按钮中心 Y 坐标
- `REPAY_Y = 860`，还款按钮中心 Y 坐标
- `STATUS_X = 260, STATUS_Y = 1000`，债务状态文字位于按钮上方
- 按钮尺寸：220 × 64（基础分辨率下）

## 9. 测试要点

- 新开一局后遗物包含 `Omamori`，不再包含 `Courier`
- 非最终幕商店可连续贷款 5 次，累计获得 `500` 金币
- 第 6 次贷款失败
- 有债务时显示 `还款` 按钮
- 金币足够时可逐次还款，每次 `100`
- 金币不足时还款失败
- 最终幕商店不允许新增贷款
- 带债进入心脏战时获得：
  - 力量 `-10`
  - 敏捷 `-10`
  - 每回合失去 `10` 生命
- 无债进入心脏战时不触发惩罚

## 实现边界

本次不做以下内容：

- 不做贷款状态的存档持久化
- 不做复杂美术资源或按钮贴图
- 不修改普通战斗、事件或地图逻辑
- 不新增完整关键词、本地化包或 BaseMod 注册体系
